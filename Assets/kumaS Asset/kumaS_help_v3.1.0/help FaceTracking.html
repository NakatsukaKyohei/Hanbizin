Q.再生モードにしたらUnityがクラッシュするのですが？

A.仕様です。

というのは半分冗談で、メモリをかなりハードに使っているので特に負荷のかかる起動時はどうしても不安定となります。 成功するまでUnity立ち上げ直して頑張ってください。

unsafe をオフにしてください。かなり確率が減るはずです。

メガネかけてる人は開始時はメガネ外してみるとクラッシュしにくくなるぞ。(わけわからん)

エラーの詳細を知りたい方は「C -> ユーザー -> (ユーザー名) -> Temp -> Unity」の中にログがあります。

これでもC#側で対処できるものはしたんだよ…。ネイティブのコードは複雑すぎてわからんかった(泣)。


Q.再生してみたけどちゃんと動かん…調整もわからん…

A.一にデバッグ、二にデバッグ、三にデバッグです。とにかくどういう値を吐き出しているか調べるのが重要です。

まずは再生をやめた時のコンソールを見てみましょう。そうすると、「suc = *, fail = *: *」というものが見つかるはずです。 sucは顔の検出に成功したフレーム数で、:の後ろのやつは検出成功率です。どうですか？


Q.!empty()というメッセージが出て、なにも動きませんが…?

A.OpenCvのカスケードファイルや、Dlibのlandmarkファイルへのパス名がおかしいです。日本語が含まれているのもダメなので、パス名に日本語の含まれていないところにファイルを移してもう一度やってみてください。


Q.どうやらまともに検出できていないみたいですが？

A.それはあなたの顔が…うわ！何をする！ｱ----ｯｯｯ!!!

偽物は消しました。ご安心を。主に考えられるのは、全体の光量が少なすぎ/多すぎではありませんか？ もしくは、背景色が肌の色と似ていませんか？というところです。この二つは、Debug options の Face Image をオンにすると確かめられます。

※注：Face image のデバッグを空のゲームオブジェクトで実行すると失敗します。Planeにアタッチして確かめましょう。


Q.検出はできているみたいだけど動きません。

A.Dobug options の Positon and Rotation をオンにしてどのような値が吐き出されているか確認しましょう。

大抵の原因は、安定した推測値が中心から離れている((0,0,0)から遠い)、または、 speed limit 系が低すぎるのどちらかです。

OpenCV・Dlib5 の時はそれに加え、感度が大きく違うというのも加わります。

感度が大きく異なる場合は Scale を、 z の position が大きく異なる時は z_offset を調整して下さい。

安定した推測値が中心から離れている場合は、安定した推測値を Position center に入力すればよいです。


Dlib68・Mixed の時、私と頭の立体座標が異なるためうまく推定できないというケースがあります。 (ほんとは皆やったほうがいいのですが)最終手段として頭の立体座標をとるというのも考えておいたほうがいいかもしれません。 なお私はレゴを頭にぶっ刺して測りました。

頭の位置を測定したらここのように設定してください


Q.デバッグが大量に出ててどうすればいいかわかりません。

A.そういう時は log to file をオンにしましょう。すべてのデバッグデータをUnityのプロジェクト直下に .csv 形式で書き出してくれます。なお、できたファイルの中身をここに記しておきます。

  • FPS_LOG ・・・FPSが書き込まれています。
  • POS_ROT_LOG ・・・推定した Position と Rotation が書き込まれています。
  • EYE_ROT_LOG ・・・推定した目の Rotation が書き込まれています。
  • EYE_RATIO_LOG ・・・推定された目の縦横比が書き込まれています。(目の開き具合)
  • FINAL_LOG ・・・いろいろ処理された後の最終的な値が書き込まれています。

これらを分析するときは、おかしな値が入っていることがあるので、(Excelの場合だと)フィルターで除去してやったほうがいいです。それから、線グラフにしてやると分析しやすいです。

Dlib68・Mixed の推定の特性上 z が-180付近になることが多々あり、それに伴い Position がよく符号逆転してます。そのため、グラフにする前にデータを加工したほうが良いです。


Q.頭がとんでもないとこにふっ飛ぶんですが!?

A.Positon offset と Position range (radius) を見直しましょう。


Q.頭がぐるぐる回ります。 Target model の設定をどうすればいいかわかりません。

A.Target model の設定をしましょう。最終的な位置の値は推定された値に Traget model の位置の値とオフセットの値を足したものとなります。 VRM、Live2D 共にモデルの一番上のオブジェクトを指定すればたいてい大丈夫です。


Q.動く感度はいいのだけどなんかむっちゃ震えてる!?

A.speed limit 系が高すぎるか Smoothness of movement が低すぎるかのどちらかです。

speed limit 系を低くしすぎると(特に速い動きだと)動かなくなり、 Smoothness of movement が高すぎると反応が遅くなります。


Q.Smoothなんとかってどうすればいいの？

A.まず、Smoothig methodの方はどちらでもいいです。どちらも似たようなものなので。詳しく知りたかったら、ググれば出てきます。LPFはローパスフィルタです。

Smoothness of movement の適性の値は初期値付近(Average:16、LPF:0.85)です。Averageは4未満、LPFは0.5未満にするとあまり意味がなくなるので気をつけてください。


Q.あの…胴がぐるんぐるん回っているのですが…(VRMのみ)

A.頭の位置がちょっと低いです。 Psition center か Position offset を調整しましょう。


Q.目の開き具合がおかしいのですが。

A.Debug options の Eye openness ratio を見て、目を開けているときの値を Eye ratio high に、目を閉じているときの値を Eye ratop low に入れるとよくなります。

また、.csvで出力させたEye openness ratio (EYE_RATIO_LOG内)を折れ線グラフにし、8移動平均線を追加するとどのような値を入れればいいのかわかりやすいです。


Q.目玉がどうやら動いていないみたいです。

A.メガネをかけている場合諦めてください。Dlibを使ってメガネをかけている人の特徴点の学習をさせるという解決策もありますが。(機械学習です。)

そうでなければ、 Eye center ratio の値を分析してください。私の場合は最大で0.1ぐらいしか動きませんでした。

全く変な値が出ている場合は、 Debug options の Eye image をオンにして目の周りの画像がどう撮れているか、また、どのようなところが検出されているか確かめてください。

よさそうな値が出ている場合は eye rotation ensitivity の値を大きくしてください。大きく動くようになります。


Q.Debug の Eye image をオンにして再生したら Unity が落ちた/無茶苦茶重くなった!

A.設定が間違っています。

Copy Cube には生成した素の Cube を、 Parents には空の GameObject を入れてください。 Cube の位置はどうでもいいですが、 Parents は以下の位置に配置しておくことを勧めます。

  • Left color (0, 0, 0)
  • Left gray (0, 0, 1)
  • Right color (-50, 0, 0)
  • Right gray (-50, 0, 1)

なお、生成されたものは Parents の子として生成されるので、要らない画像は Parents に設定した GameObject を非アクティべートにすることで見えなくすることができます。

Q.やっぱりこのプログラム重いです。もっと軽くなりませんか？

A.最適化可能です。調整するパラメータは Work threads と Resolution ついでに unsafe です。

Work threads は処理を何個に分けてやるかということで、基本数字が大きいほどパフォーマンスが上がりますが、大きすぎるとパフォーマンスが落ちます。だいたいCPUのスレッド数が一番パフォーマンスが出ます。

Resolution は画像を何分の一にして処理するのかというので、数字が大きいほど軽くなります。しかし、大きすぎると正確さが落ちます。

unsafe は並列化しちゃまずいところも並列化するかということです。ボトルネックとなっているので選択可能としました。警告が出ますが、まあそういうことです。立ち上げに成功したら、ほぼ落ちることはありません。なお、ポインタではありません。

改造をしやすいように私の作った物の説明を軽くします。

このアセットは大きく FaceTracking 部分と Apply 部分に分かれています。 FaceTraking は Webカメラから位置・回転・目の開き具合・目玉の位置を推定し、その値を保持します。 そして、 Apply はその FaceTracking の情報を読み取り、それぞれの形式に合うように変換します。

なお、 FaceTracking 部分についてはかなりやばいことになっているのでいじらないことを推奨します。(腕に自信があればどうぞご自由に改造してください)

で、その FaceTracking 部分の利用法を説明します。

クラス名は FaceTracking です。まんまです。で、利用できるのが、

  • Vector3 Position ・・・ 推定された頭の位置。
  • Quaternion Rotation ・・・ 推定された頭の回転。
  • float LeftEyeCloseness ・・・ 推定された左目の閉じ具合。0～1。0が開き。
  • float RightEyeCloseness ・・・ 推定された右目の閉じ具合。0～1。0が開き。
  • Vector3 LeftEyeRotation ・・・ 推定された左目の目玉の回転。
  • Vector3 RightEyeRotation ・・・ 推定された右目の目玉の回転。

ここらへんです。実際に使う際は、 FaceTracking を参照してそのプロパティを読み取って使ってください。あと、これらの更新タイミングは Update() の時です。
